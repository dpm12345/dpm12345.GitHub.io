ubuntu 下 arm-linux 编译环境的搭建

这学期的一门《嵌入式系统课程设计》, 需要进行交叉编译，由于依托于实验室的环境，且实验室的计算机系统为 windows，编译需在虚拟机进行，但少些文件又在 windows 系统下进行

由于以上带来的不便，便突发奇想在自己电脑上配置环境，使编写、编译、烧写均在一个系统上，提高效率

# 环境

Ubuntu16.04LTS (一开始使用的为 Ubuntu22，但由于烧写程序 Minitools 需要 QT4，而且不管怎么找解决方法也解决不了，最终还是换了系统解决的)

# 所需文件

# arm-linux-gcc 交叉编译器安装包

由于使用的是 64 位系统，故安装 64 位的编译器

链接：https://pan.baidu.com/s/1OS53_UhrAKPA692IwksaqQ
提取码：2vft

# MiniTools

为烧写程序，为方便操作，这里直接下载 Linux 版，该文件在 FriendlyElec 官网上有

点击搜索框下的 DOWNLOADS --> 下拉选择 Tiny6410 --> 选择下载渠道 (这里我选择 OneDrive，网盘限速，但用这个需要点魔法) --> 点击 FriendlyARM-Tiny6410-DVD-A --> 点击 tools --> 下载 MiniTools-Linux-20150528.tgz

(这里不选择 windows 的又一个原因为，windows10 安装时会提示驱动安装失败，需手动安装，而这个安装要将强制数字签名关闭，显得麻烦且不安全)

# Superboot-6410.bin

烧写程序所需的文件，同样可在上面提到的官网上找到

前面同上面，路径为 Tiny6410/FriendlyARM-Tiny6410-DVD-B/images/Superboot-6410.bin
