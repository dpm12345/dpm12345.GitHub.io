# 过滤命令

  • ip

    • ip.src eq / == x.x.x.x 对源地址的过滤

    • ip.dst eq / == x.x.x.x 对目标地址的过滤

    • ip.addr eq / == x.x.x.x 对源或目标的过滤

  • port

    • tcp.port eq 80 or udp.port eq 80

    • tcp.dstport == 80 只显示 tcp 协议的目标端口 80

    • tcp.srcport == 80 只显示 tcp 协议的源端口为 80

    • tcp.port >=1 and tcp.port <= 80 显示一段范围内 tcp 协议的端口

  • 过滤协议

    • tcp/udp/http...

  • 过滤 MAC

    • eth.dst == A0:00:00:04:C5:84 过滤目标 MAC

  • 包长度过滤

    • udp.lenth == 26 指 udp 本身固定长度 8 加上 udp 下面那块数据包之和

    • tcp.len >= 7 指的是 ip 数据包 (tcp 下面那块数据), 不包括 tcp 本身

    • ip.len == 94 除了以太网头固定长度 14 ，其他都是 ip.len ，及从 ip 本身到最后

    • frame.len == xxx 整个数据包长度，从 eth 开始到最后

  • http 模式过滤

    • http.request.method == "GET"

    • http.request.method == "POST"

    • http.request.url == "/img/logo-edu.gif"

    • http contains "GET"

    • http contains "HTTP/1."

    • http contains "flag"

    • http contains "KEY"

    • tcp contains "flag"

# 鼠标流量

鼠标数据包的数据长度是 4 个字节

第一个字节代表按键，当取 0x00 时，代表没有按键；当 0x01 时，代表按左键，为 0x02 时，代表按右键

第二个字节代表左右偏移，值为正，代表右移多少像素，为负，则向左移多少像素

第三个字节代表上下偏移，上为正，下位负

  • 文件 -> 导出分组解析结果 -> 为 csv，导出保存成一个.csv 文件

    tshark -r usb1.pcap(流量文件) -T fields -e usb.capdata > usbdata.txt
