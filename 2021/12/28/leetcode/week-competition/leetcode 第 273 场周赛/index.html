# 反转两次的数字

题目链接: 反转两次的数字

这道题比较简单，可以直接暴力进行两次数字反转得到答案

但仔细想想会发现，如果一个数字反转两次后是元素，那么原数字末尾必不为 0 (0 除外) ，因为如果存在 0 时 在进行反转后 0 会被删去，数字总位数减少，从而再次进行反转得到的数字必然不会是原来的数字

以下为代码

class Solution {
public:
    bool isSameAfterReversals(int num) {
        return num==0||num%10!=0;
    }
};

# 执行所有后缀指令

题目链接: 执行所有后缀指令

按照题目要求模拟即可，注意什么时候是行，什么时候是列即可

代码如下

class Solution {
public:
    vector<int> executeInstructions(int n, vector<int>& startPos, string s) {
        vector<int>ans(s.size());
        for(int i=0;i<s.size();i++)
        {
            int j=0,num=0;
            vector<int>cur=startPos;
            for(j=i;j<s.size();j++)
            {
                if(s[j]=='L'&&cur[1]>0)
                {
                    cur[1]--;
                    num++;
                }
                else if(s[j]=='R'&&cur[1]<n-1)
                {
                    cur[1]++;
                    num++;
                }
                else if(s[j]=='U'&&cur[0]>0)
                {
                    cur[0]--;
                    num++;
                }   
                else if(s[j]=='D'&&cur[0]<n-1)
                {
                    cur[0]++;
                    num++;
                }
                else
                    break;
            }
            ans[i]=num;
            
        }
        return ans;
        
    }
};
